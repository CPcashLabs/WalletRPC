# 非功能需求（NFR）

## NFR-001 性能
- 无明确业务价值时，禁止自动强制刷新。
- 同类 RPC 请求应去重、节流或缓存。
- 交易轮询应具备退避与终止策略。

## NFR-002 稳定性
- 本地存储恢复必须按 key 独立容错。
- 上下文切换（链/账户/Safe）不能残留错误状态。

## NFR-003 安全
- 助记词/私钥在导入成功后不应长期驻留在输入状态。
- 失败信息不泄露敏感上下文。

## NFR-004 可测试性
- 每个关键链路必须有单元测试。
- 主流程必须有 UI/E2E 自动化覆盖。

## NFR-005 可维护性
- 需求变更必须先更新文档。
- 代码实现应可追溯到需求与故事 ID。
