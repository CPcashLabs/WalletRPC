# 需求变更流程

## 1. 变更触发
满足任一条件即触发变更：
- 新增功能。
- 修改现有流程或业务规则。
- 调整性能、安全、稳定性目标。

## 2. 变更步骤
1. 在 `02-requirements/` 更新需求条目。
2. 在 `03-user-stories/` 增补或修订故事与验收条件。
3. 若涉及术语/状态变化，更新 `05-domain/`。
4. 在 `04-roadmap/` 调整版本计划。

## 3. 变更准入检查
- 是否说明业务价值。
- 是否定义边界条件。
- 是否具备可执行的验收标准。
- 是否识别潜在风险（安全/性能/体验）。

## 4. 完成定义（Definition of Done）
- 文档已更新且可追踪到需求 ID。
- 代码实现与测试通过。
- 回归测试覆盖关键链路。
